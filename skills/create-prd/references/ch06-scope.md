# 第6章 项目范围 — 生成指引

## 章节目标

明确项目涉及的系统、产品、接口和影响范围。

## 生成结构

```md
## 6、项目范围

### 6.1 涉及系统
| 系统名称 | 关系类型 | 影响描述 | 责任方 |
| --- | --- | --- | --- |
| {本系统} | 主体 | 新建/改造 | |
| {关联系统A} | 数据来源 | 需对接接口 | |
| {关联系统B} | 数据消费方 | 需提供接口 | |

### 6.2 影响范围
- **用户影响**：{哪些用户群体会受到影响}
- **流程影响**：{哪些现有业务流程会变化}
- **数据影响**：{是否涉及数据迁移、格式变更}
- **上下游影响**：{对上下游系统或业务的影响}

### 6.3 不在本期范围内
1. {排除项1及排除理由}
2. {排除项2及排除理由}
```

## 生成规则
1. 从用户的业务描述中推断可能涉及的系统。
2. "不在本期范围内"很重要 — 明确边界比扩大范围更有价值。

## 质量标准
- 所有涉及系统都已列出，含责任方
- 影响范围覆盖用户、流程、数据三个维度
- 有明确的"不在范围内"清单
